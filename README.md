## Project Description
This week was focused on mastering the essential tools used by professional developers: the Command Line Interface (CLI) and Git version control. I learned how to manage files, automate tasks with shell scripts, and use a professional Git branching workflow to manage my portfolio project.

## Technologies Used
- Command Line (Terminal/PowerShell)
- Git (Version Control)
- GitHub (Remote Repositories)
- Bash/PowerShell Scripting
- Markdown

## Features
- **Terminal File Management:** Created and organized the entire project structure using only CLI commands (`mkdir`, `touch`, `cp`, `mv`).
- **Automation Scripts:** Developed a project setup script (`new-project.sh`) to automate the creation of standardized folder structures.
- **Git Branching Workflow:** Implemented a feature-branch workflow, creating separate branches for specific updates and merging them into the main branch.
- **Merge Conflict Resolution:** Successfully practiced identifying and resolving code conflicts during the branch merging process.
- **Remote Collaboration:** Connected the local repository to GitHub, practicing `push`, `pull`, and `fetch` operations.

## Lessons Learned
**Efficiency through CLI:** I realized that performing file operations in the terminal is often much faster than using a GUI once you memorize the basic commands.

**Importance of Version Control:** Git is a lifesaver. Being able to "time travel" back to previous versions of my code gives me the confidence to experiment without fear of breaking the project.

**Clean Commit History:** I learned that writing meaningful commit messages (e.g., feat: add contact form) makes it much easier for other developers to understand the project's progress.

## Challenges Faced
**Merge Conflicts:** My first merge conflict was confusing. I had modified the same line in two different branches. I solved it by manually editing the file to keep the correct changes and then finalizing the merge with a new commit.

**Scripting Permissions:** On Mac/Linux, my automation script wouldn't run initially. I had to learn how to use chmod +x to make the script executable.
